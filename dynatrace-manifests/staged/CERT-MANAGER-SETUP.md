# Dynatrace Operator Webhook Certificate Fix - Implementation Guide

## Problem Summary
The webhook pods were failing to start because they were waiting indefinitely for certificates that were never provisioned. This prevented the webhook endpoint from becoming available, which blocked the K8s API server from routing admission webhook requests to it, ultimately preventing DynaKube resource creation.

## Solution: cert-manager Integration
Implemented automated certificate lifecycle management using cert-manager v1.14.0 with self-signed certificates. cert-manager automatically generates, injects, and renews TLS certificates for the webhook.

## Changes Made

### 1. **New Files Created**

#### `01-cert-manager-install.yaml`
- Installs cert-manager v1.14.0 components
- Includes: Namespace, CRDs, RBAC, Controller Deployment
- Minimal configuration suitable for production

#### `02-webhook-certificate.yaml`
- Creates `ClusterIssuer` for self-signed certificates
- Creates `Certificate` resource that generates TLS keypair
- cert-manager automatically creates secret: `dynatrace-webhook-certs`
- Certificate valid for 90 days, renews 15 days before expiry

#### `apply-staged-with-cert-manager.sh`
- Complete deployment automation script
- Applies manifests in correct order
- Includes health checks and waits between steps
- Handles cert-manager readiness before deploying webhook

### 2. **Modified Files**

#### `40-workloads-webhooks.yaml`
Three critical changes:

**A. Pod Template Annotation**
```yaml
annotations:
  dynatrace.com/inject: "false"
  kubectl.kubernetes.io/default-container: webhook
  cert-manager.io/inject-ca-from: dynatrace/dynatrace-webhook  # NEW
```

**B. Certificate Volume Mount** (lines ~280-284)
```yaml
# OLD: emptyDir volume waiting for certs that never came
volumes:
  - emptyDir:
      sizeLimit: 10Mi
    name: certs-dir

# NEW: Uses secret generated by cert-manager
volumes:
  - name: certs-dir
    secret:
      secretName: dynatrace-webhook-certs
      defaultMode: 420
```

**C. Webhook Configuration CA Bundle**
```yaml
# MutatingWebhookConfiguration metadata
annotations:
  cert-manager.io/inject-ca-from: dynatrace/dynatrace-webhook  # NEW

# ValidatingWebhookConfiguration metadata
annotations:
  cert-manager.io/inject-ca-from: dynatrace/dynatrace-webhook  # NEW
```
These annotations instruct cert-manager to inject the CA certificate bundle into the `caBundle` field automatically.

## Troubleshooting

### Issue: Certificate not generating
```bash
# Check cert-manager logs
kubectl logs -n cert-manager -l app.kubernetes.io/name=cert-manager

# Common cause: ClusterIssuer not ready
kubectl get clusterissuer dynatrace-selfsigned-issuer
# Must show: Ready=True

# Solution: Ensure cert-manager pod is running first
kubectl wait --for=condition=Ready pod -l app.kubernetes.io/name=cert-manager -n cert-manager --timeout=60s
```

### Issue: Webhook pods still not ready
```bash
# Check for certificate volume issues
kubectl describe pod -n dynatrace -l app.kubernetes.io/component=webhook | grep -A 5 "Volumes"

# Check pod resource usage (might be OOMKilled)
kubectl top pods -n dynatrace -l app.kubernetes.io/component=webhook

# Check for image pull errors
kubectl describe pod -n dynatrace -l app.kubernetes.io/component=webhook | grep -A 3 "Events"
```

### Issue: Service has no endpoints
```bash
# Check if pods have matching selector labels
kubectl get pods -n dynatrace -l app.kubernetes.io/name=dynatrace-operator,app.kubernetes.io/component=webhook -o wide

# Verify service selector
kubectl get svc dynatrace-webhook -n dynatrace -o jsonpath='{.spec.selector}'

# Check for network policies blocking
kubectl get networkpolicies -n dynatrace
```

```
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐

                           ↓
         ┌─────────────────────────────────┐

         │  ┌─────────────────────────────┤
         │  │ Volume: certs-dir (secret)  │ ← Certificate mounted here
         │  │ ├─ tls.crt                  │   ├─ From: dynatrace-webhook-certs
         │  │ └─ tls.key                  │   ├─ Generated by: cert-manager
         │  │                             │   └─ Valid: 90 days, renews at 75 days
         │  │ Container: webhook-server   │
         │  │ --certs-dir=/tmp/k8s-...    │ ← Points to mounted secret
         │  │ WEBHOOK_PORT=8443           │
         │  └─────────────────────────────┤
         └─────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│              cert-manager (cert-manager NS)             │

│  │ Certificate: dynatrace-webhook                      │
│  │ ├─ IssuerRef: dynatrace-selfsigned-issuer          │
│  │ ├─ SecretName: dynatrace-webhook-certs             │
│  │ ├─ Renews: 15 days before expiry                   │
│  │ └─ Status: Ready (auto-watches for expiry)         │
│  │                                                     │
│  │ ClusterIssuer: dynatrace-selfsigned-issuer         │
│  │ ├─ Type: Self-Signed                               │
│  │ └─ Generates certificates without external CA      │
│  └─────────────────────────────────────────────────────┤
└─────────────────────────────────────────────────────────┘
```

## Key Benefits

✅ **Automatic Certificate Management**: No manual cert provisioning required
✅ **Automatic Renewal**: Certificates renew 15 days before expiry
✅ **Self-Signed**: No need for external CA or certificate authority
✅ **Production Ready**: Minimal, secure configuration
✅ **Easy to Monitor**: Can track certificate expiry with `kubectl get certificate`
✅ **Cluster-Wide**: ClusterIssuer can be reused for other webhooks
